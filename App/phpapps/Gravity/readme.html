About

The Gravity Board X project is an open source message board system trying to be like no others.  Gravity Board X features a unique front page layout, and pushes the envelope with technologies not often found in free message boards such as a CSS-based layout, graphical WYSIWYG message editor and AJAX framework.  Please enjoy the result of the free time used to create this project, and spread the word if you enjoy it!

1. Installation

Requirements/General Information

In order to be compatible with the GBX AJAX framework, browsers must support JavaScript and the XMLHttpRequest Object.  Popular browsers supporting this include: Mozilla Firefox 1.0, Internet Explorer 5.0, Opera 7.6, Netscape 7.1, and Safari 1.2 or any later versions of these browsers.  You are discouraged to use any version of Internet Explorer prior to version 7, as previous versions do not fully comply with the W3C CSS standards.

The following software has been tested with Gravity Board X v2.0 and should be fully compatible:

  • PHP 4.3.11

  • PHP 5.0.4 and 5.0.5

  • MySQL 4.0.25-standard

  • Mozilla Firefox v1.5.0.6 or later

Important: Although the most recent version of Microsoft Internet Explorer (version 7) is much better with its CSS support, we still do not believe it is at the point it should be.  Therefore, Gravity Board X was developed and tested mainly with Mozilla Firefox and designed to be fully compliant with CSS2 according to the W3C standard.  Very minimal testing was conducted using Internet Explorer.  Because of this, the current release of GBX neither supports or recommends the use of Microsoft Internet Explorer (version 7 or earlier) with this software, and by all means, please do not use version 6 or earlier.  If you must, use version 7 as it has the best CSS support to date.  For the best compatibility, we recommend that you use Mozilla Firefox.  Mozilla Firefox is available for download here, and is compatible with Windows, Linux and Mac.

Gravity Board X has been validated as XHTML 1.0 Transitional and CSS2 according to W3C standards.  If you find otherwise or encounter any errors, please contact admin@gravityboardx.com so that the errors may be fixed.

Installation

Installation of Gravity Board X is quick and simple.  Follow these easy steps to install the software on your web server.

1. Extract the .zip file to a directory on your local computer.
2. Upload the files contained within the .zip file to a directory on your web server.  Keep the entire directory structure in tact.  The following files should not be uploaded: uninstall.php, readme.html
3. CHMOD the directory where you uploaded the files to rwerwerew (777).  If this is not done, the last part of installation will fail.
4. Navigate your browser to install.php in the directory where you uploaded the files to.  Example: http://yoursite.com/gbx/install.php
5. Follow the simple steps of the installer which will guide you through the rest of the setup process.
6. After installation, remove the file install.php from your web server.

2. Release Notes

What's new this release

  • GBX is now fully compliant if the php ini settings have register_globals set to OFF.  It also does not give any errors or notices if E_ALL is enabled, however, E_ALL is disabled by default.
  • Switched WYSIWYG editors from htmlArea to FCKeditor - more widely accessible by different browsers and more easily customizable.  FCKeditor brings an unprecedented message editor to Gravity Board X, not seen in most free message board systems.  Creating a Word-like environment for posting messages brings a whole new dimension to the message board.  Easily modify text, insert images, create hyperlinks, add smileys, and format your message.  No more BBcode, you see each change happen as you make it.  Oh yeah, you can print directly from the editor, too.
  • Implemented site-wide input validation system.  Open source project yav is used to validate all user input on the client side using JavaScript.
  • Member List feature added
  • Censored words expanded and optimized (smaller size)
  • User agreement feature added
  • gZip compression added for all pages, when supported by the browser
  • Fully restructured Administrator Control Panel
  • Debugger added: The debugger is meant to primarily be used during development and should have no reason to be used for any other purpose. It should NOT be turned on while people are visiting the board.
  • Added a "Who's Online" page to the board navigation
  • Restructured board footer
  • Passwords are now 128-bit MD5 one-way encrypted in the database and are not able to be decrypted for any reason whatsoever.  Forgotten passwords will be reset.
  • Email validation is required for a user to login after registration (a valid email address is required to sign up)
  • txt readme upgraded to a fancy state-of-the-art next-gen HTML document (oooo aaaahhh)
  • Restructured the entire site layout and upgraded to a CSS based layout, instead of the prior HTML table layout.  No more tables, they are a thing of the past.  The only time GBX will ever use tables again is if information needs to be displayed in a tabular format.  However, this will always be avoided at all costs.  The new look is ideal for browser windows between 800 and 1280 pixels.
  • Information tags added to board pages.  If a thread contains an image, hyperlink, or is empty, a small image is displayed next to the thread title signifying IMG, URL, or EMP, respectively.
  • Board descriptions added to the main page
  • Added "Forgot My Password" and "Resend Validation Email" features.  "Forgot My Password" assigns the user a new password via email (as passwords are unrecoverable for security)
  • An uncountable number of small and large bug fixes and tweaks
  • A new look: GBX has dropped its heavily graphical header in favor of a much more simple design.  This will be easier to skin and has a "Web 2.0" look.
  • AJAX framework has been implemented across the board.  Use it to do great things such as reorder boards with drag and drop or post and edit messages on the fly.
  • Cookies are now supported to remember users when they come back


FEATURES NOT YET IMPLEMENTED BUT PLANNED FOR FUTURE RELEASES:

-Message logging. This makes it so when a user views a post, it is recorded, and in the "New" table column, it shows the number of unread messages in that post or board.**

**Signifies a feature partially implemented, but disabled for this release.
